# OctoAcme — Onboarding Checklists for New Roles

## Purpose
Provide structured onboarding checklists for new team members joining in the following roles: Release Manager, UX Designer, Technical Writer, Stakeholder Champion, Security Lead, and Support Engineer.

---

## Release Manager Onboarding Checklist

### Week 1: Orientation
- [ ] Review [Roles and Personas](octoacme-roles-and-personas.md) and [Release & Deployment Guide](octoacme-release-and-deployment.md)
- [ ] Meet with Project Manager and Product Manager to understand current release cadence
- [ ] Get access to deployment pipelines and release tooling
- [ ] Review recent release notes and rollback procedures
- [ ] Shadow a release cycle with a current team member

### Week 2: Hands-On
- [ ] Coordinate a deployment window with the team
- [ ] Document any gaps in release process documentation
- [ ] Participate in pre-release readiness review
- [ ] Connect with Support Engineer for post-release handoff process

### Ongoing
- [ ] Own release scheduling and go/no-go decisions
- [ ] Maintain release notes and deployment documentation
- [ ] Participate in retrospectives to share release insights

---

## UX Designer Onboarding Checklist

### Week 1: Orientation
- [ ] Review [Roles and Personas](octoacme-roles-and-personas.md) and [Project Planning](octoacme-project-planning.md)
- [ ] Meet with Product Manager to understand current product priorities
- [ ] Get access to design tools and component libraries
- [ ] Review existing user research and design documentation
- [ ] Shadow a design review or feedback session

### Week 2: Hands-On
- [ ] Conduct a walkthrough of the current product experience
- [ ] Identify initial opportunities for UX improvements
- [ ] Participate in backlog refinement for user-facing features
- [ ] Connect with Developers on design feasibility discussions

### Ongoing
- [ ] Own design reviews and user experience validation
- [ ] Collaborate on accessibility and usability testing
- [ ] Participate in retrospectives to share design learnings

## Technical Writer Onboarding Checklist

### Week 1: Orientation
- [ ] Review [Roles and Personas](octoacme-roles-and-personas.md) and all docs in this folder
- [ ] Meet with Project Manager and Release Manager to understand documentation cadence
- [ ] Get access to documentation repositories and publishing tools
- [ ] Review recent release notes and user-facing documentation
- [ ] Identify key subject matter experts for each area

### Week 2: Hands-On
- [ ] Audit existing documentation for gaps and inconsistencies
- [ ] Contribute to an upcoming release note or documentation update
- [ ] Establish a documentation review workflow with Developers and QA
- [ ] Connect with Support Engineer to understand common user pain points

### Ongoing
- [ ] Own documentation updates aligned with release cycles
- [ ] Maintain onboarding materials and process docs
- [ ] Participate in retrospectives to capture process improvements

## Security Lead Onboarding Checklist

### Week 1: Orientation
- [ ] Review [Roles and Personas](octoacme-roles-and-personas.md) and [Risk Management & Communication](octoacme-risks-and-communication.md)
- [ ] Meet with Project Manager and Developers to understand security posture
- [ ] Get access to security scanning tools and incident response channels
- [ ] Review recent security reviews and vulnerability reports
- [ ] Familiarize yourself with compliance requirements and security standards

### Week 2: Hands-On
- [ ] Conduct a security review of a current feature or release
- [ ] Identify gaps in security documentation or runbooks
- [ ] Participate in a pre-release readiness review
- [ ] Connect with Release Manager on security gate processes

### Ongoing
- [ ] Own security reviews and threat modeling sessions
- [ ] Drive remediation of security vulnerabilities
- [ ] Participate in retrospectives and post-incident reviews

## Support Engineer Onboarding Checklist

### Week 1: Orientation
- [ ] Review [Roles and Personas](octoacme-roles-and-personas.md) and [Release & Deployment Guide](octoacme-release-and-deployment.md)
- [ ] Meet with Release Manager and Product Manager to understand support handoff process
- [ ] Get access to support ticketing system and customer communication channels
- [ ] Review recent incidents and customer feedback reports
- [ ] Shadow a support triage session

### Week 2: Hands-On
- [ ] Respond to a customer issue with guidance from a mentor
- [ ] Document a bug reproduction and escalate to Developers
- [ ] Collect and summarize customer feedback for Product team
- [ ] Connect with Technical Writer on support documentation needs

### Ongoing
- [ ] Own post-release support and incident triage
- [ ] Provide customer insights to inform product improvements
- [ ] Participate in retrospectives to share support learnings

## General Onboarding Tips

- Review the [Project Management Overview](octoacme-project-management-overview.md) for context on OctoAcme's project lifecycle
- Attend team standups and planning sessions to understand team rhythm
- Ask questions and provide fresh perspective—your onboarding experience helps improve our processes
- Document any unclear areas and share feedback with your onboarding buddy
